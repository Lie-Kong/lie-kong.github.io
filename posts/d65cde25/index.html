使用Cloudflare拯救被墙的小鸡

Created2025-05-25|Updated2025-05-25|有趣的网络
|Post Views:|Comments:

前言

CloudFlare CDN 以免费著称，因此， CloudFlare 也有了赛博大善人的外号。CloudFlare Tunnel 是 CloudFlare推出的一款免费隧道软件，用于将内网端口暴露在公网上(即内网穿透)，我们可以借助 Tunnel 将我们的服务与互联网连接。

经测试 VLESS/Vmess 均可使用此方法，本文以Vmess作为演示，Vless同理，仅需更改协议即可

警告：
本文仅供个人学习交流，请不要用于非法目的，请不要在生产环境中使用。

前置条件

  • 一个托管在Cloudflare的域名
  • 一台vps
  • 一个具有阅读能力的大脑

这里我使用Ducky家的arm nat和某小网站购买的年抛域名jfipq0.tech作为演示

部署

安装3X-UI

市面上3x-ui的脚步一大堆，我在这里使用的是MHSanaei大佬的3x-ui
安装脚本：bash <(curl -Ls https://raw.githubusercontent.com/MHSanaei/3x-ui/refs/tags/v2.6.0/install.sh)
在运行脚本以后，会提示你 Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]: 这里的意思是是否要进行自定义配置，可以进行用户名和密码的自定义设置等，这里输入n来简化安装过程
安装完成后，屏幕上会输出管理员信息，例如：

This is a fresh installation, generating random login info for security concerns:
###############################################
Username: lbaGPB2XxP
Password: DTcCx9iaOj
Port: 6220
WebBasePath: a4w5ELxIPaMZ5mC
Access URL: http://140.238.239.36:6220/a4w5ELxIPaMZ5mC
###############################################

记住 Port Username Password，稍后会用到

创建Cloudflare tunnel

  1. 登录到Cloudflare Dashboard，点击左侧的Zero Trust (注:部分地区需魔法才能打开)
  2. 输入团队名称
    团队名称可任意起名，系统会验证是否重名。注册 Cloudflare 后第一次登陆，进入 Zero Trust 时会看到下面的页面。

    如果已经添加过团队名称会看到下面的页面
  3. 选择免费计划(Free $0 / user / month)
    输入团队名称后，点击 Next 进入付费计划选择页面，选免费计划。
  4. 刷新网页,回到主页面
  5. 点击左侧的Networks - Tunnels
  6. Creat a tunnel,选择 Cloudflared

    随意输入一个隧道名称

    根据页面提示完成安装,安装后,页面下方"Connectors"内会显示连接的设备,点击NEXT
  7. 配置tunnel
    Subdomain 任意,Domain 选择你托管在cloudflare的域名,Type 选择http,URL 输入 localhost:+ 安装3x-ui时输出的Port,我这里是6220
    例如,这样:

配置3x-ui

在浏览器访问上一步设置的域名再加上安装3x-ui时输出的WebBasePath
例如,我的链接就是https://dci-br.jfipq0.tech/a4w5ELxIPaMZ5mC/,然后就可以看到熟悉的界面了

登陆后,点击左侧’入站列表’,添加入站,协议选择vmess 传输选择 WebSocket,如图配置

点击创建,记下端口号,回到Cloudflare Tunnel的配置界面,点击Add a public hostname,重复第7点的操作,将Port改为入站的端口号
最终效果如下图所示,总共两条隧道,一条用于管理面板,一条用于连接代理

至此,网页端已经配置完成,可以添加客户端配置了
