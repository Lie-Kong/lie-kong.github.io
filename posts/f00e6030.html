前言

该文章用于记载一些本人在服务器部署过程中遇到的一些离谱的，网上无法直接搜到的问题与解决方案

文章不定时更新

1.Docker 无法启动

错误现象：docker安装完成后，启动报错docker: Cannot connect to the Docker daemon at unix:///var/run/docker.sock. Is the docker daemon running?
复现方法：重装纯净Debian12后使用官方脚本安装docker
解决方法：sudo groupadd docker #创建一个名为docker的组
sudo usermod -aG docker $USER #将当前用户加入 docker 组

2.Sudo报错

错误现象：安装sudo后,使用sudo命令,提示sudo: unable to resolve host XXXXX: Name or service not known,但命令可正常执行
复现方法：安装纯净Debian12后使用apt安装sudo,使用任意sudo开头的命令
解决方法: vi /etc/host
插入 127.0.0.1 XXXXX
